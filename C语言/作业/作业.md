## 作业:分析下面代码的结果
```c
unsigned char a = 255;
char b = 255;  
printf("%d %u\n",a,a); 
printf("%d %u\n",b,b); 

unsigned short a = -1; 
int b = a; 
printf("%d\n",a);
printf("%d\n",b);


unsigned char a = -1;  
unsigned int b = -1; 
printf("%d %u\n",a,b);
```

对于无符号字符型变量a ，它的取值范围是0 ~ 255

对于字符型变量b，它的取值范围是-128 ~ 127

`printf("%d %u\n",a ,a);`

中，已知对于整型数据%d，打印出来的结果为255 而对于%u（四字节十进制整数）来说，因为符号位提升到了一个大尺寸类型，所以对于多余的位全部置为0，打印出来的结果也为255.

`printf("%d %u",b ,b);`
中，因为b为字符型变量，取值范围为-128 ~ 127 所以对其赋值255溢出，255- (127+1) -128  = -1
那么打印的结果应该为-1。 而在四字节十进制整数中来说，因为符号位为1，那么多余位数全部置为1，得到的结果应该为1111 1111 1111 1111 1111 1111 1111 1111 ，结果值为4294967295

  printf("%d\n", b);
  //对于整型变量b 因为上面的无符号短整型的值为65535，所以b的值也为65535
